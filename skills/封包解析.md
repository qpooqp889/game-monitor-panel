# 封包解析：setAuto（自動戰鬥技能設定）

> 記錄遊戲截獲的 `setAuto` 事件格式與用途

---

## 觸發範例

以下為 WebMonitor 面板實際截獲的封包：

```
[WB-SEND] setAuto [{"openSkill":"sk_fireball"}]    ← 開怪技能
[WB-SEND] setAuto [{"atkSkill":"sk_fireball"}]    ← 攻擊技能
```

---

## 封包結構

```
Socket.IO EVENT: setAuto
Payload: Array[Object]
  └─ 單一物件，key 為技能類型，value 為技能 ID 字串
```

### 格式圖解

```
emit('setAuto', [
  {
    "openSkill": "sk_xxxxx"    ← 開怪技能，進入戰鬥後第一個施放
  }
])

emit('setAuto', [
  {
    "atkSkill": "sk_xxxxx"     ← 攻擊技能，戰鬥中循環施放
  }
])
```

---

## 欄位說明

### `openSkill` — 開怪技能（戰鬥開場）

| 項目 | 說明 |
|------|------|
| 觸發時機 | 玩家進入戰鬥模式後、**第一個**施放的技能 |
| 用途 | 先手控制/爆發（如火球暈眩、冰凍等） |
| 施放次數 | 每場戰鬥只施放一次 |
| 常見值 | `sk_fireball`、`sk_freeze`、`sk_poison` 等 |

### `atkSkill` — 攻擊技能（自動普攻）

| 項目 | 說明 |
|------|------|
| 觸發時機 | 開怪技能施放後，**持續循環**施放 |
| 用途 | 主要輸出技能 |
| 施放次數 | 每場戰鬥可施放多次（取決於冷卻與 MP） |
| 常見值 | `sk_fireball`、`sk_slash`、`sk_arrow` 等 |

---

## 與 `castSkill` 的差異

| 比較項目 | `castSkill` | `setAuto` |
|---------|------------|-----------|
| 用途 | 手動單次施放 | 設定自動戰鬥配置 |
| 參數 | `{id: "skillId"}` | `[{openSkill:"sk_xxx"}]` 或 `[{atkSkill:"sk_xxx"}]` |
| 觸發次數 | 呼叫幾次就放幾次 | 依遊戲自動判斷（開場一次 / 循環） |
| 可否同時設定 | 否 | 可同時傳送 `openSkill` + `atkSkill` |
| 範例 | `emit('castSkill',{id:'strike'})` | `emit('setAuto',[{atkSkill:'sk_fireball'}])` |

---

## 傳送方式

### 透過 `__wbSocket`（遊戲面板攔截的 Socket.IO 實例）

```javascript
// 設定開怪技能（火球）
window.__wbSocket.emit('setAuto', [{openSkill: 'sk_fireball'}]);

// 設定攻擊技能（火球）
window.__wbSocket.emit('setAuto', [{atkSkill: 'sk_fireball'}]);

// 兩者同時設定（推薦）
window.__wbSocket.emit('setAuto', [
  {openSkill: 'sk_fireball'},
  {atkSkill: 'sk_fireball'}
]);
```

### 安全性檢查

```javascript
if (window.__wbSocket && window.__wbSocket.connected) {
  window.__wbSocket.emit('setAuto', [{atkSkill: 'sk_fireball'}]);
} else {
  console.warn('[GM] Socket 未連線，無法發送 setAuto');
}
```

## 遊戲狀態中的呈現

`window.lastState`（從 `state` Socket.IO 事件取得）可能包含：

```javascript
window.lastState = {
  char: { hp: 800, maxHp: 1000, mp: 200, maxMp: 500, ... },
  mode: 'combat',        // 戰鬥模式
  monsters: [
    { n: '惡魔士兵', hp: 500, maxHp: 500, lv: 12 },
    { n: '惡魔士兵', hp: 500, maxHp: 500, lv: 12 }
  ],
  // 視伺服器實作而定，可能有 autoSkills / openSkill / atkSkill 等欄位
};
```

## 實用範例

### 範例 1：自動設定火球術為攻擊技能

```javascript
// 在 Console 中執行
window.__wbSocket.emit('setAuto', [{atkSkill: 'sk_fireball'}]);
```

### 範例 2：設定開怪 + 攻擊（推薦做法）

```javascript
window.__wbSocket.emit('setAuto', [
  {openSkill: 'sk_fireball'},   // 進戰鬥先手
  {atkSkill: 'sk_fireball'}     // 之後普攻
]);
```

### 範例 3：先進規則（AdvFarm）自動化

在 `advanced-farming.js` 的規則引擎中：

```javascript
// 規則：進入戰鬥時，自動設定火球術為開怪技能
{
  name: '火球術開怪',
  enabled: true,
  cooldownMs: 5000,
  conditions: [
    {type: 'monsterCount', op: '>', value: '0'}
  ],
  actions: [
    {type: 'setAutoOpen', skillId: 'sk_fireball'}
  ]
}
```

> **注意**：上方的 `setAutoOpen` 為 AdvFarm 擴展，未來可能實作。目前建議直接呼叫：
> ```javascript
> window.__wbSocket.emit('setAuto', [{openSkill: 'sk_fireball'}]);
> ```

## 注意事項

1. **`setAuto` 只傳送一次**：伺服器接收後會自動套用，不需要每幀重送
2. **同時只能有一組配置**：後送的 `setAuto` 會覆蓋先前的設定
3. **`openSkill` 與 `atkSkill` 可分開傳**：遊戲應會合併處理
4. **技能 ID 大小寫敏感**：`sk_fireball` ≠ `SK_FIREBALL`
5. **需在戰鬥狀態下設定**：部分遊戲可能限制非戰鬥時的 `setAuto` 行為
