## 封包來源

本擴充套件支援**兩個獨立的封包通道**：

| 通道 | 來源 | 優先級 | 用途 |
|------|------|--------|------|
| **Socket.IO** | `window.io.Socket.prototype` (onevent / packet / emit) | 🥇 主 | Boss、技能、狀態、自動控制 |
| **WebSocket** | `WebSocket.prototype.send` + `.addEventListener('message')` | 🥈 備 | 早期封包日誌、兼容舊版 |

### 啟動順序

```javascript
// 1. 立即檢查並安裝 SIO hook（每 300ms 重試）
setTimeout(installSioHook, 500);

// 2. 同時掛 WebSocket 層（保留舊 packets 數組）
var origSend = WebSocket.prototype.send;
WebSocket.prototype.send = function(data){ ... };
```

## SIO Hook 機制

### Hook 對象

```javascript
var SP = window.io && window.io.Socket && window.io.Socket.prototype;
```

### Hook 1: `SP.packet` (SEND 攔截)

```javascript
SP.__wbPkt = true;
var _origPkt = SP.packet;
SP.packet = function(packet) {
  var type = packet && packet.type;       // 2 = EVENT, 3 = ACK
  if (type === 2 || type === 3) {
    var ev   = packet.data && packet.data[0] || null;
    var args = packet.data && packet.data.slice(1) || [];
    window.__wbBossEmitLog.push({
      t: Date.now(),
      dir: 'SEND',
      evt: ev,
      args: JSON.stringify(args).slice(0, 300)
    });
    console.log('[WB-SEND]', ev, JSON.stringify(args).slice(0, 150));
  }
  return _origPkt && _origPkt.call(this, packet);
};
```

**捕獲的封包**:
- `bossAction` - 世界王按鍵
- `usePotion` - 使用藥水
- `castSkill` - 施放技能
- `setBossSet` - 設定 Boss set
- `setZone` - 切換地圖
- `attack` - 攻擊
- `toLobby` - 返回大廳
- `viewInv` - 查看背包
- `sortInv` - 排序背包
- `probe` - 探測
- `selectChar` - 選擇角色
- `enterGame` - 進入遊戲

### Hook 2: `SP.onevent` (RECEIVE 攔截)

```javascript
SP.__wbOE = true;
var _oe = SP.onevent;
SP.onevent = function(p) {
  if (!window.__wbSocket) window.__wbSocket = this;
  if (p && p.data && p.data[0]) {
    var evtName = p.data[0];
    window.__sioPackets.push({
      t: Date.now(),
      dir: 'EVENT',
      evt: evtName,
      args: JSON.stringify(p.data).slice(0, 500)
    });
    // 關鍵：解析 state 事件 → window.lastState
    if (evtName === 'state' && p.data[1]) {
      window.lastState = p.data[1];
    }
  }
  if (_oe) _oe.call(this, p);
};
```

**捕獲的事件**:
- `state` - 角色完整狀態（HP/MP/EXP/位置/怪物清單等）⭐
- `onlineCount` - 在線人數
- `mail` - 郵件
- `market` - 市場
- `worldBosses` - 世界王列表
- `marquee` - 跑馬燈
- `chat` - 聊天

## WebSocket Hook 機制

### Hook Send

```javascript
var origSend = WebSocket.prototype.send;
WebSocket.prototype.send = function(data) {
  window.__battleStatus.packets.push({ type: 'send', data: data });
  window.__ws = this;
  return origSend.call(this, data);
};
```

### Hook Receive (延遲 100ms 確保 ws 已建立)

```javascript
setInterval(function() {
  if (window.__ws && !window.__ws.__hooked) {
    window.__ws.__hooked = true;
    window.__ws.addEventListener('message', function(e) {
      window.__battleStatus.packets.push({ type: 'receive', data: e.data });
      try {
        var d = JSON.parse(e.data.substring(2));
        if (d[0] === 'onlineCount') window.__gmOnlineCount = d[1];
      } catch (err) {}
    });
  }
}, 100);
```

**重要**: WebSocket 封包格式是 `42[...]` (Socket.IO Engine.IO 協議)，需要 `.substring(2)` 去掉前綴才能 `JSON.parse`。

## 封包發送 (Send)

### 統一介面 (v2.02)

所有發送都**優先使用 SIO emit**，降級到 WebSocket：

#### `sendZone(zoneId)`
```javascript
function sendZone(zoneId) {
  if (window.__wbSocket && window.__wbSocket.connected) {
    window.__wbSocket.emit('setZone', zoneId);
    console.log('[GM] Teleport (SIO):', zoneId);
  } else if (window.__ws && window.__ws.readyState === 1) {
    window.__ws.send('42["setZone","' + zoneId + '"]');
    console.log('[GM] Teleport (WS):', zoneId);
  }
}
```

#### `sendCmd(cmd)`
```javascript
function sendCmd(cmd) {
  if (window.__wbSocket && window.__wbSocket.connected) {
    window.__wbSocket.emit(cmd);
    console.log('[GM] Cmd (SIO):', cmd);
  } else if (window.__ws && window.__ws.readyState === 1) {
    window.__ws.send('42["' + cmd + '"]');
    console.log('[GM] Cmd (WS):', cmd);
  }
}
```

### 為什麼要保護 WebSocket readyState?

WebSocket 的 `readyState`：
- `0` CONNECTING
- `1` OPEN ✅
- `2` CLOSING
- `3` CLOSED ❌

在 `readyState !== 1` 時呼叫 `.send()` 會**直接踢出遊戲**。v2.02 加入了 `readyState === 1` 檢查避免此問題。

## 故障排除

### Q1: 面板出現但 `__wbSocket` 為 null

**原因**: SIO hook 還沒抓到 socket 實例。

**解法**:
1. 確認 SIO hook 已安裝：`__wbSioHooked === true`
2. 在 Console 執行 `window.io()` 手動取得 socket
3. 重新整理頁面讓 hook 重新掛載

### Q2: 點地圖按鈕遊戲跳出

**原因**: WebSocket 已關閉但 `send()` 還在執行。

**解法**: v2.02 已加入 `readyState === 1` 保護，請確認使用最新版。

### Q3: Boss 按鈕 disabled 灰色

**原因**: `updateBossCd` 鎖定了按鈕。

**解法**: 開啟 BOSS Tab 的「🛡️ 跳過冷卻」開關，會自動 patch `updateBossCd` 和 `renderBossPanel`。

### Q4: 看不到 state 事件

**原因**: onevent hook 沒抓到。

**檢查**:
```javascript
// 在 Console 執行
console.log(window.__wbSioHooked);   // 應該是 true
console.log(window.__sioPackets.length);  // 應該 > 0
```

### Q5: 自動戰鬥時打不到怪物

**原因**: 目標索引錯誤。

**解法**: 自動戰鬥邏輯已修正，從 `state.monsters[].hp > 0` 找目標。
